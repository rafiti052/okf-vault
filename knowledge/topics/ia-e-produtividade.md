# IA e Produtividade

Tema recorrente em 6 dos 8 notes do corpus. A tese central emergente: ganhos individuais de produtividade são reais mas não sobem automaticamente para a organização — o redesenho de processo é o ingrediente ausente.

## Tensão central

[ROI em IA](../notes/roi-em-ia.md) documenta que 70% das empresas não veem ROI apesar de super users economizarem 9h/semana. [O Novo Dev](../notes/o-novo-dev.md) explica que o plateau de 30-40% persiste quando a IA é usada apenas na fase de codificação. [Cloud Code norms](../notes/cloud-code-norms.md) mostra o que acontece quando o processo inteiro é redesenhado — codificar deixa de ser o gargalo.

## Notas neste tópico

- [Cloud Code team norms](../notes/cloud-code-norms.md) — o que muda quando coding não é mais o bottleneck; JIT planning, protótipos vs. docs, shift-left
- [O Novo Dev](../notes/o-novo-dev.md) — plateau de 30-40% e como superá-lo com IA em todo o pipeline
- [ROI em IA](../notes/roi-em-ia.md) — dados empíricos: custo crescente, ROI travado no indivíduo, 8 meses para retorno
- [Agentic AI](../notes/agentic-ai.md) — métricas devem ser definidas pelo core business; caso Elfa (3x vendas)
- [The Missing Interface for AI](../notes/missing-interface-ai.md) — automação de processo ruim não gera produtividade; fricção vs. valor
- [Adoção de arquitetura](../notes/adocao-arquitetura.md) — custo de abstração reduz eficácia da IA; estratégia Opus-para-planejar/Sonnet-para-implementar
